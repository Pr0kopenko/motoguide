# Как править мотогайд с телефона

Репозиторий: <https://github.com/Pr0kopenko/motoguide>

## Самый простой способ

1. Откройте репозиторий в мобильном браузере и войдите в GitHub.
2. Найдите нужный файл и нажмите значок карандаша **Edit this file**.
3. Внесите изменение.
4. Нажмите **Commit changes** и сохраните правку прямо в ветку `main`.
5. Откройте вкладку **Actions**. Зелёная галочка у запуска «Проверка проекта» означает, что сайт собирается и тесты проходят.

## Что где менять

- [`app/page.tsx`](https://github.com/Pr0kopenko/motoguide/edit/main/app/page.tsx) — названия маршрутов, километраж, время, описания, остановки, предупреждения и источники.
- [`app/globals.css`](https://github.com/Pr0kopenko/motoguide/edit/main/app/globals.css) — цвета, размеры, отступы и мобильный дизайн.
- [`app/layout.tsx`](https://github.com/Pr0kopenko/motoguide/edit/main/app/layout.tsx) — заголовок и описание страницы для браузера и поисковиков.
- [`docs/PRODUCT.md`](https://github.com/Pr0kopenko/motoguide/edit/main/docs/PRODUCT.md) — план, продуктовая защита, критерии приёмки и тест-кейсы.
- [`README.md`](https://github.com/Pr0kopenko/motoguide/edit/main/README.md) — описание проекта и ссылки.

Для большой правки удобнее открыть <https://github.dev/Pr0kopenko/motoguide> — это браузерный редактор всего проекта. На маленьком экране точечная правка через карандаш обычно проще.

## Как обновить опубликованный сайт

Коммит в GitHub сохраняет и проверяет изменение, но текущий адрес на `chatgpt.site` сам не обновляется. После правки напишите Codex:

> Подтяни последние изменения из GitHub, проверь и опубликуй новую версию мотогайда.

Codex загрузит правку, прогонит проверки и обновит публичный сайт. Для полностью автоматической публикации без Codex потребуется подключить отдельный хостинг к репозиторию, например Cloudflare Pages или Vercel.
